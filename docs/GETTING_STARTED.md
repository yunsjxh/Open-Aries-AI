# Open-Aries-AI 快速上手

这份文档面向第一次使用 Open-Aries-AI 的用户。目标是先把程序跑起来、配置好模型，然后完成一个简单任务。

## 这个项目能做什么

Open-Aries-AI 是 Windows 桌面自动化助手。你用自然语言描述目标，它会读取当前屏幕或控件信息，让大模型规划步骤，并自动执行点击、输入、打开应用、读写文件、运行命令等操作。

常见用法：

- 打开应用，例如记事本、计算器、浏览器。
- 在网页或软件中搜索、点击、填写内容。
- 创建、读取、修改本地文件。
- 连接 MCP 工具，让模型调用外部服务。

## 推荐使用方式

如果你只是想使用软件，优先运行 Web GUI 版本：

```powershell
.\aries_web.exe
```

程序启动后，浏览器访问：

```text
http://localhost:8080
```

如果你更习惯命令行，也可以运行：

```powershell
.\aries_agent.exe
```

## 第一次启动

你需要准备一个可用的大模型 API Key。项目默认提供智谱 AI 配置，也支持 OpenAI 兼容接口。

### Web GUI

1. 运行 `aries_web.exe`。
2. 打开 `http://localhost:8080`。
3. 进入“设置”页面。
4. 选择已有提供商，或添加新的 OpenAI 兼容提供商。
5. 填写 API Base URL、模型名称和 API Key。
6. 回到“任务执行”页面，输入任务目标。

### 命令行

1. 运行 `aries_agent.exe`。
2. 按提示输入 API Key。
3. 如果要使用自定义提供商，输入 `provider`。
4. 程序询问模型是否支持视觉时：
   - 支持图片识别的模型选 `y`。
   - 纯文本模型选 `n`。
5. 输入任务目标。

## 选择模型模式

### 视觉模式

适合支持图片输入的模型。程序会截图，把屏幕图片发给模型分析。

推荐用于：

- 网页操作。
- 桌面软件点击。
- 需要理解界面布局的任务。

### 纯文本模式

适合不支持图片输入的模型。程序会优先使用 UI Automation 控件树；也可以配置一个视觉模型先把屏幕转述成文字。

推荐用于：

- 控件结构清晰的 Windows 应用。
- 不想让主模型处理图片的场景。
- 需要节省视觉模型成本的场景。

## 试一个简单任务

建议先从低风险任务开始：

```text
打开记事本，输入“你好，Open-Aries-AI”
```

或者：

```text
打开计算器，计算 123 + 456
```

任务开始后，程序会先生成执行计划，然后逐步执行。Web GUI 会显示计划、日志和任务状态；命令行版本会同时显示桌面悬浮状态窗。

## 常用命令

命令行版本支持这些特殊输入：

| 命令 | 作用 |
| --- | --- |
| `provider` | 切换或配置 AI 提供商 |
| `key` | 打开智谱 AI API Key 页面 |
| `mcp` | 管理 MCP 服务器 |
| `update` | 检查更新 |
| `clear` | 清除保存的 API Key 和配置 |
| `quit` / `exit` | 退出程序 |

## MCP 怎么用

MCP 是可选能力。没有 MCP，Open-Aries-AI 也可以完成基础桌面自动化任务。

Web GUI：

1. 打开“MCP 服务器”页面。
2. 选择 `stdio` 或 `streamable_http`。
3. 填写服务器名称和启动命令或 URL。
4. 连接成功后，模型可以在任务中调用 MCP 工具。

命令行：

```text
mcp
```

然后按菜单提示连接、查看或断开 MCP 服务器。

## 安全提示

Open-Aries-AI 可以操作本机桌面和文件，请先用简单任务验证配置。

使用时建议：

- 不要让模型处理不确定的删除、覆盖、批量移动文件任务。
- 文件写入优先使用用户目录或项目目录。
- 避免让模型写入 `C:\Windows`、`C:\Program Files` 等系统目录。
- 对执行命令、写文件、删除文件等高风险操作保持确认。

v1.3.2 已加入系统路径写入保护、FileList 重复调用检测和结构化日志，但用户仍应确认任务目标是否安全。

## 常见问题

### 程序启动后浏览器打不开

手动访问：

```text
http://localhost:8080
```

如果端口被占用，先关闭已有的 `aries_web.exe`，再重新启动。

### API Key 验证失败

检查三项：

1. API Key 是否复制完整。
2. API Base URL 是否以服务商文档为准。
3. 模型名称是否存在，并且当前账号有权限调用。

### 模型看不懂界面

优先尝试视觉模式。如果使用纯文本模型，可以配置一个视觉模型做屏幕转述。

### 任务一直重复同一步

停止当前任务，重新描述目标，并把路径、窗口名或期望结果写得更明确。v1.3.2 已对重复列目录等行为做了拦截，但清晰目标仍能显著提升成功率。

### 需要看详细日志

查看程序目录下的日志文件：

```text
aries_agent.log
```

Web GUI 中也可以在“系统日志”页面查看实时日志。

## 从源码编译

如果你只想使用软件，直接运行 Release 中的 exe 即可。需要自己编译时，使用 MinGW-w64 UCRT64：

```bash
g++ -std=c++17 -O2 -mconsole -static aries_agent.cpp -o aries_agent.exe \
    -lgdiplus -lgdi32 -lws2_32 -lshell32 -ladvapi32 -lcrypt32 \
    -lpthread -lole32 -loleaut32 -luuid -lwininet

g++ -std=c++17 -O2 -mconsole -static web_main.cpp -o aries_web.exe \
    -lgdiplus -lgdi32 -lws2_32 -lshell32 -ladvapi32 -lcrypt32 \
    -lpthread -lole32 -loleaut32 -luuid -lwininet
```
